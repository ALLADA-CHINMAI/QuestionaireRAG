## Getting Started

### 1. Clone the repo
```bash
git clone https://github.com/ALLADA-CHINMAI/QuestionaireRAG.git
cd QuestionaireRAG
```

### 2. Install dependencies
```bash
pip3 install -r requirements.txt
```
> Note: `sentence-transformers` will download the `all-MiniLM-L6-v2` model (~90MB) on first run. This is a one-time download cached locally.

### 3. Get a free Groq API key
- Sign up at **console.groq.com** (free)
- Go to API Keys в†’ Create Key
- Copy the key (starts with `gsk_...`)

### 4. Set up environment
```bash
cp .env.example .env
# Open .env and set:
# GROQ_API_KEY=gsk_...
```

### 5. Add your data
```
data/
в”њв”Ђв”Ђ questionnaires/     в†ђ place CAIQ XLSX here
в””в”Ђв”Ђ customers/
    в”њв”Ђв”Ђ customer_1/     в†ђ SOP + SOW PDFs for customer 1
    в”њв”Ђв”Ђ customer_2/     в†ђ SOP + SOW PDFs for customer 2
    в””в”Ђв”Ђ customer_3/     в†ђ SOP + SOW PDFs for customer 3
```

### 6. Start the server
```bash
python3 -m uvicorn main:app --reload
```
> If `uvicorn` is not on PATH, always use `python3 -m uvicorn` instead of `uvicorn` directly.

### 7. Build the index (one-time)
```bash
curl -X POST http://localhost:8000/index/questionnaires
```

### 8. Query for a customer
```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"customer_id": "customer_1", "top_k": 20}'
```

### 9. Interactive API docs
Open **http://localhost:8000/docs** in your browser to explore and test all endpoints via Swagger UI.

## Architecture (v2)

### Indexing Pipeline (run once)
1. Parse CAIQ XLSX в†’ extract `question_id`, `question_text`, `domain` per row
2. Embed each question в†’ dense vector via HuggingFace `all-MiniLM-L6-v2` (local)
3. Build BM25 index over all question texts (sparse/keyword)
4. Persist ChromaDB collection to disk + save BM25 index

### Query Pipeline (per customer request)
1. Upload customer SOP/SOW PDFs
2. Parse PDFs via PyMuPDF в†’ extract full text
3. LLM (Groq вЂ” `llama-3.1-8b-instant`) summarizes key topics/context from combined docs
4. Embed the summary в†’ query vector
5. Run **hybrid search**:
   - Dense: ChromaDB cosine similarity в†’ top-K
   - Sparse: BM25 keyword match в†’ top-K
6. **Reciprocal Rank Fusion (RRF)** merges both ranked lists
7. Return final ranked list of questions with metadata

## Tech Stack (v2)

| Layer | Tool | Notes |
|-------|------|-------|
| XLSX Parsing | `openpyxl` | CAIQ questionnaire |
| PDF Parsing | `PyMuPDF (fitz)` | Customer SOP/SOW docs |

| Dense Embeddings | `all-MiniLM-L6-v2` (HuggingFace вЂ” local, free) | 384-dim vectors |
| Vector Store | `ChromaDB` (local, persisted) | Dense search |
| Sparse / Keyword | `rank_bm25` | BM25 index |
| Hybrid Fusion | Reciprocal Rank Fusion (RRF) | Merges dense + sparse |
| API Framework | `FastAPI` + `uvicorn` | REST endpoints |
| Config | `python-dotenv` | API keys via .env |

## API Endpoints (v2)

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/index/questionnaires` | Index all questionnaires from XLSX |
| `POST` | `/query` | Upload customer PDFs, get ranked questions |
| `GET`  | `/health` | Health check |

### POST /query вЂ” Request
```json
{
  "customer_id": "customer_1",
  "top_k": 20
}
```
Reads PDFs from `data/customers/{customer_id}/` automatically.

### POST /query вЂ” Response
```json
{
  "customer_id": "customer_1",
  "total_results": 20,
  "questions": [
    {
      "rank": 1,
      "question_id": "IAM-01.1",
      "domain": "IAM",
      "question": "Are identity and access management policies...",
      "rrf_score": 0.032,
      "source": "CAIQv4.0.3"
    }
  ]
}
```
