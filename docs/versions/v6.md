# QuestionaireRAG — Architecture & Flow

**Version:** 6.0  
**Date:** 2026-04-15

---

## System Architecture

### **Three Main Workflows**

1. **SOP Upload & Indexing** — Index Standard Operating Procedures with capability tags
2. **Questions Upload & Indexing** — Index custom questions with categories
3. **SOW Query** — Upload Statement of Work files → Get ranked questions

## Core File Responsibilities

### **1. main.py**
**Purpose:** Application entry point  
**Flow:**
- Initializes FastAPI app
- Includes routes from `app/api/routes.py`
- Mounts static file serving
- Starts uvicorn server

---

### **2. app/api/routes.py**
**Purpose:** HTTP endpoint handlers  
**Flow:**
- `POST /query-sow` — Main query endpoint
  - Validates uploaded SOW files (.docx, .pdf, .xlsx)
  - Saves to temp directory
  - Calls `retrieve_for_sow()` from retriever.py
  - Formats response as `SowQueryResponse`
  - Cleans up temp files

- `POST /index/upload-sops` — Index SOP documents
  - Receives SOP files + capability labels
  - Calls `build_sop_index()` from indexer.py

- `POST /index/upload-questions` — Index questions
  - Receives questions Excel file
  - Calls `build_questions_index()` from indexer.py

---

### **3. app/core/ingestor.py**
**Purpose:** Document parsing & chunking  
**Flow:**
- `parse_document(file_path)` → Extracts text from .docx, .pdf, .xlsx
  - Uses `python-docx` for Word documents
  - Uses `PyPDF2` for PDFs
  - Uses `openpyxl` for Excel files
  
- `chunk_text(text, chunk_size=300, overlap=75)` → Token-based chunking
  - Splits text into ~400 token chunks with 100 token overlap
  - Preserves sentence boundaries
  
- `load_sow_file(file_path)` → SOW-specific loader
  - Returns list of chunk dicts with metadata
  - Structure: `{chunk_id, filename, chunk_text, chunk_index}`

---

### **4. app/core/embedder.py**
**Purpose:** Generate embeddings & compute similarity  
**Flow:**
- `embed_texts(texts: List[str])` → Batch embedding
  - Calls Azure OpenAI `text-embedding-ada-002` model
  - Returns list of 1536-dim float vectors
  - Uses Azure AD token authentication with caching
  
- `embed_query(text: str)` → Single embedding
  - Wrapper around `embed_texts()` for single query
  
- `compute_cosine_similarity(vec1, vec2)` → Similarity score
  - Computes cosine similarity between two vectors
  
- `compute_similarity_matrix(capabilities, categories)` → Semantic mappings
  - Creates N×M matrix of capability ↔ category similarities
  - **Note:** Currently not called due to missing logger initialization

---

### **5. app/core/azure_search.py**
**Purpose:** Azure Cognitive Search client wrapper  
**Flow:**
- `search_questions_hybrid(query_vector, query_text, top=20)`
  - Executes hybrid search (vector + BM25)
  - Searches `question_items` index
  - Returns ranked question documents
  
- `search_sop_hybrid(query_vector, query_text, top=5)`
  - Searches `sop_chunks` index
  - Returns SOP chunks with capability tags
  
- `index_sop_chunks(documents)` → Upload SOP chunks to Azure
- `index_psmart(documents)` → Upload questions to Azure

---

### **6. app/core/indexer.py**
**Purpose:** Build & manage Azure search indices  
**Flow:**
- `build_sop_index(sop_files: List[tuple])`
  - Parses SOP files via `ingestor.load_sop_file()`
  - Embeds all chunks via `embedder.embed_texts()`
  - Uploads to Azure `sop_chunks` index
  
- `build_questions_index(xlsx_path: str)`
  - Parses Excel file via `ingestor.load_questions_xlsx()`
  - Embeds each question
  - Uploads to Azure `question_items` index
  
- `load_questions_store()` / `load_psmart_questions_store()`
  - Loads cached question metadata from JSON
  - Used for enriching results with full question text

---

### **7. app/core/retriever.py**
**Purpose:** Query orchestration & ranking logic  
**Flow:**
- `retrieve_for_sow(sow_file_paths, top_n=20)`
  
  **Step 1:** Parse SOW files
  ```python
  chunks = load_sow_file(path)  # via ingestor
  ```
  
  **Step 2:** Embed chunks
  ```python
  embeddings = embed_texts(chunk_texts)  # via embedder
  ```
  
  **Step 3:** Dual-path search
  ```python
  # Path 1: Direct SOW → Questions
  direct = azure_client.search_questions_hybrid(embedding, chunk_text)
  score *= SOW_DIRECT_BOOST  # 1.2× boost
  
  # Path 2: SOW → SOP → Questions
  sop_matches = azure_client.search_sop_hybrid(embedding, chunk_text)
  for sop in sop_matches:
      capability = sop["capability"]
      questions = azure_client.search_questions_hybrid(
          embedding, 
          f"{chunk_text} {capability}"
      )
      # Base score - no boost (1.0×)
  ```
  
  **Step 4:** Deduplicate by `question_id`, keep highest score
  
  **Step 5:** Sort & limit to `top_n`
  
  **Step 6:** Enrich with metadata
  ```python
  q_meta = questions_store.get(question_id)
  result = {
      "rank": rank,
      "question_id": qid,
      "category": q_meta["category"],
      "question": q_meta["question_text"],
      "score": final_score,
      "match_path": "Direct SOW match" | "SOW → SOP (capability)",
      "sow_context": first_150_chars,
      "sow_filename": filename,
      "sop_context": sop_chunk if applicable,
      "sop_capability": capability if applicable
  }
  ```

## Ranking Algorithm & Score Calculation

### **Score Components**

```python
# Base score from Azure Cognitive Search hybrid search
base_score = hybrid_search_score  
# (Combination of vector cosine similarity + BM25 keyword matching)

# Path-based boost application
if match_path == "Direct SOW match":
    final_score = base_score * SOW_DIRECT_BOOST  # 1.2× boost
else:  # "SOW → SOP (capability)"
    final_score = base_score * 1.0  # No boost

# Deduplication: if same question_id appears in both paths,
# keep the one with higher final_score

# Sort all candidates by final_score descending
# Take top_n results
```

### **Score Range**

- **Base score range:** Typically 0.0 to 1.0 (from Azure CS hybrid search)
  - Cosine similarity component: -1.0 to 1.0 (usually positive: 0.0-1.0)
  - BM25 component: weighted and normalized
  - Combined via Azure's proprietary scoring formula

- **After boost:** 0.0 to ~1.2 (theoretical max with 1.2× boost)
  - Direct path: `base_score × 1.2`
  - SOP-mediated path: `base_score × 1.0`

- **Observed range in practice:** 0.03 to 0.06
  - Low scores indicate weak semantic similarity
  - Higher scores (>0.5) would indicate strong relevance
  - Current results suggest moderate-to-weak matches

### **Boost Rationale**

The 1.2× boost for direct SOW matches prioritizes questions that:
- Have direct semantic overlap with SOW content
- Match specific terminology used in the client's SOW
- Align with explicit requirements stated in the SOW

The SOP-mediated path (no boost) serves to:
- Discover questions related to broader capability themes
- Fill gaps when direct matching is weak
- Provide category-based question recommendations
