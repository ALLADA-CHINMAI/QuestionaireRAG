## System Architecture

### **Three Main Workflows**

1. **SOP Upload & Indexing** — Index Standard Operating Procedures with capability tags
2. **Questions Upload & Indexing** — Index custom questions with categories
3. **SOW Query** — Upload Statement of Work files → Get ranked questions

  - Parses JSON response, maps scores back to candidates

**Purpose:** Document parsing & chunking  
**Flow:**

  - Wrapper around `embed_texts()` for single query
  
- `compute_cosine_similarity(vec1, vec2)` → Similarity score
  - Computes cosine similarity between two vectors
  
- `compute_similarity_matrix(capabilities, categories)` → Semantic mappings
  - Creates N×M matrix of capability ↔ category similarities
  - **Note:** Currently not called due to missing logger initialization

### **7. app/core/retriever.py**
**Purpose:** Query orchestration & ranking logic  
**Flow:**
- `retrieve_for_sow(sow_file_paths, top_n=20)`
  
  **Step 1:** Parse SOW files
  ```python
  chunks = load_sow_file(path)  # via ingestor
  ```
  
  **Step 2:** Embed chunks
  ```python
  embeddings = embed_texts(chunk_texts)  # via embedder
  ```
  
  **Step 3:** Dual-path search
  ```python
  # Path 1: Direct SOW → Questions
  direct = azure_client.search_questions_hybrid(embedding, chunk_text)
  score *= SOW_DIRECT_BOOST  # 1.2× boost
  
  # Path 2: SOW → SOP → Questions (NEW!)
  # 2a: Find relevant SOP chunks
  sop_matches = azure_client.search_sop_hybrid(embedding, chunk_text, top=5)
  
  # 2b: Embed SOP chunks
  sop_texts = [sop["chunk_text"] for sop in sop_matches]
  sop_embeddings = embed_texts(sop_texts)
  
  # 2c: For each SOP chunk, find related questions
  for sop_chunk, sop_embedding in zip(sop_matches, sop_embeddings):
      via_sop_questions = azure_client.search_questions_hybrid(
          sop_embedding, 
          sop_chunk["chunk_text"],
          top=8
      )
      
      # Combined score: (SOP relevance × 0.5 + Question match × 0.5) × 1.3 boost
      for q in via_sop_questions:
          combined_score = (sop_chunk["score"] * 0.5 + q["score"] * 0.5) * 1.3
          
          # Only keep if better than existing candidate
          if combined_score > existing_score[q.id]:
              candidates[q.id] = {
                  "score": combined_score,
                  "match_path": "Via SOP match",
                  "sow_context": chunk_text[:400],
                  "sop_context": sop_chunk["chunk_text"][:400]
              }
  
  # Via SOP matches can now override Direct matches due to 1.3× boost!
  ```
  
  **Step 4:** GPT-4o re-ranking
  ```python
  # Get top 50 candidates from vector search
  top_candidates = get_top_n(candidates, n=50)
  
  # Re-rank using GPT-4o
  for candidate in top_candidates:
      context = f"SOW: {sow_context}\nSOP: {sop_context}"
      question = candidate.question_text
      
      relevance_score = gpt4o_rate_relevance(context, question)
      # Returns 0-10 score
      
      candidate.final_score = relevance_score
  
  # Sort by GPT-4o scores and take top_n
  ```
  
  **Step 5:** Enrich with metadata & explanations
  ```python
  q_meta = questions_store.get(question_id)
  
  # Generate explanation using GPT-4o
  explanation = gpt4o_explain_relevance(sow_context, sop_context, question)
  
  result = {
      "rank": rank,
      "question_id": qid,
      "category": q_meta["category"],
      "question": q_meta["question_text"],
      "score": gpt4o_score,  # 0-10 scale
      "match_path": "Via SOP match",  # or "Direct SOW match"
      "sow_context": first_400_chars,
      "sow_filename": filename,
      "sop_context": sop_chunk if applicable,
      "explanation": explanation  # NEW: Why this question is relevant
  }
  ```

## Match Path Logic

### **Two Match Paths**

1. **"Direct SOW match"** (Boost: 1.2×)
   - SOW chunk → Questions Index (direct hybrid search)
   - Fast, keyword-heavy matching
   - Good for SOWs with explicit question terminology

2. **"Via SOP match"** (Boost: 1.3×) — **NEW!**
   - SOW chunk → SOP Index → Questions Index (two-hop search)
   - Stronger semantic matching through SOP intermediaries
   - Better for SOWs referencing SOP concepts/capabilities
   - **Can override Direct matches** due to higher boost

### **When Via SOP Wins**

Example: Customer2 SOW mentions "data classification using Azure Purview"
- **Direct path:** Finds generic "data" questions (weak match)
- **Via SOP path:** 
  1. Matches "Data Governance and Security" SOP (strong match)
  2. SOP chunk mentions PII/PHI classification
  3. Finds "Do you classify sensitive data?" question (strong match)
  4. Combined score with 1.3× boost **beats** direct match

### **Match Type Breakdown Logging**

retriever.py now logs:
```

📊 Match breakdown: 23 Direct SOW matches, 24 Via SOP matches
```

## Ranking Algorithm & Score Calculation

### **Two-Stage Ranking Strategy**

**Stage 1: Vector Search** (Fast, broad recall)
```python
# Base score from Azure Cognitive Search hybrid search
base_score = hybrid_search_score  
# (Combination of vector cosine similarity + BM25 keyword matching)

# Apply boost for direct SOW matches
initial_score = base_score * SOW_DIRECT_BOOST  # 1.2× boost

# Get top 50 candidates for re-ranking
```

**Stage 2: GPT-4o Re-ranking** (Deep semantic analysis)
```python
# For each of top 50 candidates:
prompt = f"""
Rate the relevance of this security question to the given SOW/SOP context on a scale of 0-10.

SOW Context: {sow_chunk}
SOP Context: {sop_chunk}
Question: {question_text}

Provide a relevance score (0-10) and brief explanation.
"""

gpt4o_response = azure_openai.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": prompt}],
    temperature=0.0
)

final_score = extract_score(gpt4o_response)  # 0-10 scale
explanation = extract_explanation(gpt4o_response)

# Sort by final_score DESC and take top_n
```

### **Score Range**

**Vector Search Scores (Stage 1):**
- **Base score range:** Typically 0.0 to 1.0 (from Azure CS hybrid search)
  - Cosine similarity component: -1.0 to 1.0 (usually positive: 0.0-1.0)
  - BM25 component: weighted and normalized
  - Combined via Azure's proprietary scoring formula

- **After boost:** 0.0 to ~1.2 (with 1.2× boost)
  - Used only for initial candidate selection
  
- **Observed range:** 0.03 to 0.06
  - Low scores indicate weak semantic similarity
  - This is why GPT-4o re-ranking is critical

**GPT-4o Scores (Stage 2):**
- **Range:** 0 to 10
  - 0-3: Low relevance (question not applicable)
  - 4-6: Moderate relevance (question somewhat related)
  - 7-8: High relevance (question directly applicable)
  - 9-10: Critical relevance (question essential for this SOW)

### **Why GPT-4o Improves Results**

**Problem with vector-only search:**
- Low scores (0.043-0.050) suggest weak semantic matching
- Embeddings capture surface-level similarity, not deep relevance
- Cannot understand business context or implicit requirements

**GPT-4o advantages:**
- Deep semantic understanding of SOW requirements
- Can infer implicit needs from context
- Understands domain-specific terminology
- Provides explainable reasoning
- Significantly better accuracy than cosine similarity alone

## Configuration & Setup

### **1. Retriever Match Boost Configuration (retriever.py)** — **NEW in v6.1**

Update these values in `app/core/retriever.py` to control match type scoring:

```python
# Direct SOW → Questions matches
SOW_DIRECT_BOOST = 1.2      # 1.2× boost for direct matches

# Via SOP matches (SOW → SOP → Questions)
SOW_VIA_SOP_BOOST = 1.3     # 1.3× boost - can override direct matches!

# Questions retrieved per SOP chunk
TOP_QUESTIONS_PER_SOP = 8   # How many questions to find per SOP match

# SOP chunks to retrieve per SOW chunk
TOP_SOP_MATCHES = 5         # How many SOPs to consider for each SOW chunk

# Direct questions per SOW chunk
TOP_DIRECT_QUESTIONS = 10   # Direct lookup limit
```

**Why Via SOP boost is higher (1.3 vs 1.2):**  
Via SOP matches go through semantic intermediaries, providing stronger conceptual relevance. This allows SOP-related questions to compete with or override keyword-heavy direct matches.

### **2. Azure OpenAI Configuration (reranker.py)**

Update these values in `app/core/reranker.py`:

```python
AZURE_OPENAI_ENDPOINT = "https://YOUR_ENDPOINT.openai.azure.com/"
```

GPT-4o re-ranking is **enabled by default** in `retrieve_for_sow()`:

## Troubleshooting

### **GPT-4o Re-ranking Fails**

**Symptom:** System falls back to vector-only scoring

**Possible causes:**
1. Azure OpenAI endpoint not configured
2. GPT-4o deployment name incorrect
3. Authentication failure
4. Rate limiting or quota exceeded
5. JSON parsing error in response

**Solution:**
- Check logs for error messages
- Verify endpoint and deployment name
- Test Azure credentials locally
- Increase timeout or retry logic
- Validate prompt generates valid JSON

### **Low Scores (Still < 1.0)**

**Symptom:** GPT-4o returns unexpectedly low scores

**Possible causes:**
1. SOW/SOP context too short or truncated
2. Questions not genuinely relevant
3. Prompt not clear enough

**Solution:**
- Increase context window in `reranker.py`
- Review prompt engineering
- Check if questions match SOW domain

### **Slow Performance**

**Symptom:** Query takes >10 seconds

**Optimization:**
- Reduce batch size (fewer questions per call, more parallel)
- Reduce candidate pool (re-rank top 30 instead of 50)
- Enable caching for repeat queries
- Consider async processing for large batches
