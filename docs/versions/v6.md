**Date:** 2026-04-15
**Status:** Enhanced RAG with SOW-Based Prioritization & Configurable Results
**Version Codename:** Next-Gen Intelligence Era

## Overview

**Version 6** introduces a comprehensive RAG enhancement featuring SOW (Statement of Work) based question prioritization, configurable result sets, and supporting context for each question. The system now supports parallel indexing of SOPs (Standard Operating Procedures), questions, and intelligent mapping between document types for superior relevance ranking.

### Key Improvements

- **SOW-Based Question Prioritization**: Questions matched to Statement of Work documents rank higher than those matched only to SOPs
- **Configurable Top-N Results**: Users can dynamically control the number of results displayed via the UI (1–N configurable range)
- **Supporting Context**: Each question includes transparency about its relevance source (which SOW/SOP chunk it relates to)
- **Multi-Document Processing**: Support for uploading multiple SOPs, SOWs, and a questions Excel file simultaneously
- **Semantic Capability Mapping**: Automatic mapping of SOP capabilities to question categories using embeddings and similarity matching
- **Enhanced Chunking**: Token-based chunking (400 tokens per chunk, 100-token overlap) for SOPs and SOWs
- **Unified Index Architecture**: Azure Cognitive Search manages all document types with rich metadata

### Breaking Changes

None. All v5 API endpoints maintain backward compatibility while supporting new features.

**`app/core/indexer.py`** — Multi-index management

```python
class AzureSearchIndexer:
    def build_sop_index(sop_documents: List[UploadFile]) -> IndexBuildReport:
        """
        1. Parse and chunk SOPs (token-based, 400/100)
        2. Tag each chunk with: capability, filename, section, metadata
        3. Generate embeddings (Ada-002, 1536-dim)
        4. Upload to Azure index: sop_chunks
        Returns: count, capabilities extracted, warnings
        """
    
    def build_question_index(questions_excel: UploadFile) -> IndexBuildReport:
        """
        1. Parse Excel: extract category + question columns
        2. For each question: generate embedding
        3. Tag with: category, excel_row_id, source_file
        4. Upload to Azure index: question_items
        Returns: count, categories extracted, warnings
        """
    
    def build_semantic_mappings(sop_chunks: List, question_categories: List) -> MappingReport:
        """
        1. Get embeddings for SOP capability keywords
        2. Get embeddings for question categories
        3. Compute similarity (cosine) between each SOP capability and Q category
        4. Store high-confidence mappings (> 0.7 similarity) in semantic_mappings index
        5. Fallback: use keyword matching if embedding similarity is low
        Returns: mapping_count, confidence_distribution
        """
```

### Phase 2: Query Flow

#### 2.1 SOW Processing & Retrieval Logic

```python
def process_sow_query(sow_files: List[UploadFile], 
                      top_n: int,
                      ranking_mode: str = "sow_priority") -> QueryResult:
    """
    1. Parse and chunk SOWs (token-based, 400/100)
    
    2. For each SOW chunk:
       a. Hybrid search against sop_chunks index (vector + BM25)
       b. Retrieve top-5 SOP chunks with ranking scores
    
    3. For each SOP chunk:
       a. Extract capability tag from SOP metadata
       b. Find question categories via semantic_mappings
       c. Query question_items index with category + SOW embedding
       d. Retrieve top-10 questions ranked by relevance to SOW
    
    4. Direct search:
       a. For each SOW chunk, hybrid search against question_items
       b. Retrieve top-20 questions ranked by semantic similarity
    
    5. Combine question results:
       a. Deduplicate by question_id
       b. Apply ranking priority (source tracking):
          - sow_priority: SOW-direct matches weighted higher
          - balanced: equal weight to SOW and SOP-derived
          - sop_priority: SOP-derived matches weighted higher
    
    6. Apply top_n limit and format response with context
    
    Return: QueryResult with ranked questions + supporting context
    """
```

## Ranking Algorithm (v6)

### Question Ranking Priority

```
For each deduplicated question:
  base_score = semantic_similarity(sow_chunk, question)
  
  if matched_via_sow_direct:
    sow_direct_score = base_score * 1.5          // 50% boost
  else if matched_via_sop_derived:
    sop_derived_score = base_score * 1.0         // baseline
  
  if matched_via_category_mapping:
    category_boost = 1.2                         // 20% boost
    sop_derived_score *= category_boost
  
  if keyword_match(sow_chunk, question):
    keyword_boost = 1.1                          // 10% boost
    final_score *= keyword_boost
  
  final_score = apply_ranking_mode(sow_direct_score, sop_derived_score)
  
  return (question, final_score)

sort by final_score DESC
apply top_n limit
```

### Ranking Modes

- **sow_priority** (default): SOW-direct matches weighted 1.5x higher
- **balanced**: Both sources weighted equally (1.0x)
- **sop_priority**: SOP-derived matches weighted 1.5x higher

## Performance Considerations

1. **Chunking Overhead**: Token-based chunking adds ~5–10% indexing time
2. **Semantic Mapping**: Initial computation is O(capabilities × categories); results are cached
3. **Query Optimization**: Multi-index queries are parallelized at Azure Cognitive Search layer
4. **Top-N Efficiency**: Limiting results on client-side after ranking (server returns all candidates)
