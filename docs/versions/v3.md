## Getting Started

### 1. Clone the repo
```bash
git clone https://github.com/ALLADA-CHINMAI/QuestionaireRAG.git
cd QuestionaireRAG
```

### 2. Install dependencies
```bash
py -m pip install -r requirements.txt
```
> **Note:** On Windows, use `py -m pip` instead of `pip` directly.

### 3. Set up Azure OpenAI credentials

You'll need access to an Azure OpenAI service with:
- **gpt-4o** deployment for LLM summarization
- **text-embedding-ada-002** deployment for embeddings
- Azure AD authentication (tenant ID, client ID, client secret)

### 4. Configure environment
```bash
cp .env.example .env
```

Edit `.env` and set your Azure credentials:
```bash
# Azure OpenAI Configuration
OPENAI_ENDPOINT=https://your-endpoint.openai.azure.com
OPENAI_DEPLOYMENT_NAME=gpt-4o
OPENAI_API_KEY=your-api-key-here
EMBEDINGS_OPENAI_DEPLOYMENT_NAME=text-embedding-ada-002

# Azure Authentication
AUTH_TENANT_ID=your-tenant-id
AUTH_CLIENT_ID=your-client-id
AUTH_CLIENT_SECRET=your-client-secret
AUTH_SCOPE=api://your-scope/.default
```

> **Security:** Never commit `.env` to git. It's already in `.gitignore`.

### 5. Add your data
```
data/
├── questionnaires/     ← place CAIQ XLSX here
└── customers/
    ├── customer_1/     ← SOP + SOW PDFs for customer 1
    ├── customer_2/     ← SOP + SOW PDFs for customer 2
    └── customer_3/     ← SOP + SOW PDFs for customer 3
```

### 6. Start the server
```bash
py -m uvicorn main:app --reload
```

### 7. Build the index (one-time)
```bash
curl -X POST http://localhost:8000/index/questionnaires
```
> **Important:** If you're migrating from v2, delete the old `data/chroma_db/` directory first, as the embedding dimensions changed from 384 to 1536.

### 8. Query for a customer
```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"customer_id": "customer_1", "top_k": 20}'
```

### 9. Interactive API docs
Open **http://localhost:8000/docs** in your browser to explore and test all endpoints via Swagger UI.

## Changes from v2

### Major Changes
- **Migrated from local HuggingFace embeddings to Azure OpenAI** (`text-embedding-ada-002`)
  - Embedding dimensions: **384 → 1536** (requires re-indexing)
  - Better semantic understanding and retrieval quality
  - API-based (no local model downloads)

- **Migrated from Groq to Azure OpenAI** for LLM (`gpt-4o`)
  - Enterprise-grade security and compliance
  - More consistent and higher quality summarization
  - Better integration with existing Azure infrastructure

- **Added Azure AD authentication**
  - Client credentials flow with token refresh
  - Automatic token caching with 5-minute buffer
  - Uses `azure-identity` library for credential management

### Breaking Changes
- **ChromaDB must be rebuilt** due to embedding dimension change
- `.env` configuration completely changed (see setup section)
- Different dependencies (`openai`, `azure-identity` instead of `sentence-transformers`, `groq`)

## Architecture (v3)

### Indexing Pipeline (run once)
1. Parse CAIQ XLSX → extract `question_id`, `question_text`, `domain` per row
2. Embed each question → **1536-dim dense vector** via Azure OpenAI `text-embedding-ada-002`
3. Build BM25 index over all question texts (sparse/keyword)
4. Persist ChromaDB collection to disk + save BM25 index

### Query Pipeline (per customer request)
1. Upload customer SOP/SOW PDFs
2. Parse PDFs via PyMuPDF → extract full text
3. LLM (Azure OpenAI **gpt-4o**) summarizes key topics/context from combined docs
4. Embed the summary → **1536-dim query vector** via Azure OpenAI
5. Run **hybrid search**:
   - Dense: ChromaDB cosine similarity → top-K
   - Sparse: BM25 keyword match → top-K
6. **Reciprocal Rank Fusion (RRF)** merges both ranked lists
7. Return final ranked list of questions with metadata

### Authentication Flow
1. Application authenticates with Azure AD using client credentials
2. Receives JWT access token (cached for reuse)
3. Token automatically refreshed when near expiration (5-min buffer)
4. Token passed in Authorization header for all Azure OpenAI API calls

| API Framework | `FastAPI` + `uvicorn` | REST endpoints |
| Config | `python-dotenv` | Credentials via .env |

## Migration Guide (v2 → v3)

### 1. Update dependencies
```bash
py -m pip install -r requirements.txt
```

New packages:
- `openai>=1.0.0` — Azure OpenAI SDK
- `azure-identity>=1.12.0` — Azure AD authentication

Removed packages:
- `sentence-transformers` — replaced by Azure OpenAI embeddings
- `groq` — replaced by Azure OpenAI LLM

### 2. Update environment variables

**Old (.env in v2):**
```bash
GROQ_API_KEY=gsk_...
```

**New (.env in v3):**
```bash
OPENAI_ENDPOINT=https://your-endpoint.openai.azure.com
OPENAI_DEPLOYMENT_NAME=gpt-4o
OPENAI_API_KEY=your-api-key-here
EMBEDINGS_OPENAI_DEPLOYMENT_NAME=text-embedding-ada-002
AUTH_TENANT_ID=your-tenant-id
AUTH_CLIENT_ID=your-client-id
AUTH_CLIENT_SECRET=your-client-secret
AUTH_SCOPE=api://your-scope/.default
```

### 3. Delete old vector database
```bash
Remove-Item -Recurse -Force data\chroma_db
```

This is required because embedding dimensions changed (384 → 1536).

### 4. Rebuild the index
```bash
curl -X POST http://localhost:8000/index/questionnaires
```

This will re-embed all questions using Azure OpenAI and create a new ChromaDB with 1536-dim vectors.

### 5. Test
```bash
# Check health
curl http://localhost:8000/health

# Run a query
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"customer_id": "customer_1", "top_k": 20}'
```

## Configuration Reference

### Azure OpenAI Deployments Required

You need two deployments in your Azure OpenAI service:

1. **Embedding Model**
   - Deployment name: `text-embedding-ada-002` (or set `EMBEDINGS_OPENAI_DEPLOYMENT_NAME`)

2. **LLM Model**
   - Deployment name: `gpt-4o` (or set `OPENAI_DEPLOYMENT_NAME`)
   - Model: `gpt-4o`

## Known Issues & Troubleshooting

### Issue: "Invalid Azure AD JWT"
**Cause:** Incorrect client secret or expired token

**Solution:**
1. Verify `AUTH_CLIENT_SECRET` is the secret **value**, not the secret ID
2. Check that the client secret hasn't expired in Azure Portal
3. Ensure `AUTH_SCOPE` matches your Azure OpenAI resource configuration

### Issue: Server fails to start
**Cause:** Missing dependencies

**Solution:**
```bash
py -m pip install -r requirements.txt
```

### Issue: "Index not built yet"
**Cause:** ChromaDB not initialized

**Solution:**
```bash
curl -X POST http://localhost:8000/index/questionnaires
```

### Issue: Git push rejected (secrets detected)
**Cause:** `.env` file committed to git

**Solution:**
```bash
# Remove from git tracking
git rm --cached .env
git commit -m "Remove .env from version control"
git push

# Note: Rotate any exposed secrets in Azure Portal
```

## Performance Notes

### Indexing Time
- **v2** (local embeddings): ~30 seconds for 263 questions
- **v3** (Azure OpenAI): ~2-3 minutes for 263 questions (API latency)
  - Batch processing helps reduce API calls
  - One-time operation, results persist to disk

### Query Time
- **Embedding query**: ~100-200ms (Azure OpenAI API call)
- **Summarization**: ~1-2 seconds (gpt-4o API call)
- **Hybrid search**: ~50-100ms (local ChromaDB + BM25)
- **Total per query**: ~2-3 seconds end-to-end

### Cost Considerations
- **Embeddings**: ~$0.0001 per 1K tokens
  - 263 questions ≈ ~$0.01-0.02 for indexing
- **LLM (gpt-4o)**: ~$0.005-0.015 per 1K tokens
  - Per query ≈ ~$0.01-0.02 depending on document size
