## Overview

**Version 4** marks a major architectural upgrade from **Chroma DB + manual HNSW/BM25** to **Azure Cognitive Search with native hybrid indexing**.

### Key Improvements

✨ **Hybrid Search**: Vector (HNSW) + BM25 keyword search in single query  
✨ **Semantic Ranking**: AI-powered relevance reranking for improved quality  
✨ **Simplified Architecture**: Single Azure CS query replaces RRF fusion logic  
✨ **Multi-tenancy**: Per-customer document indexes with strong isolation  
✨ **Production Ready**: Comprehensive error handling, logging, and monitoring  
✨ **Backward Compatible**: Same API endpoints, improved responses  

### Breaking Changes

**Minimal**: Only one new optional parameter added to `/query` endpoint

```python
# Request: New optional parameter (defaults to True)
{
  "customer_id": "customer_1",
  "top_k": 20,
  "use_semantic_ranking": true  # NEW — optional
}

# Response: One renamed field, one new optional field
{
  "questions": [
    {
      "rank": 1,
      "question_id": "IAM-01.1",
      "domain": "IAM",
      "question": "...",
      "score": 2.5,              # CHANGED: was rrf_score
      "semantic_score": 0.95,    # NEW: optional (if semantic ranking enabled)
      "source": "CAIQ"
    }
  ]
}
```

## Getting Started

### Prerequisites

- Python 3.9+
- Azure subscription (with Cognitive Search service)
- Azure OpenAI access (for embeddings & gpt-4o)

### 1. Clone & Install

```bash
git clone https://github.com/ALLADA-CHINMAI/QuestionaireRAG.git
cd QuestionaireRAG
git checkout azure-cognitive-search  # Switch to new branch (or main after merge)

# Create virtual environment
python3 -m venv venv
source venv/bin/activate  # macOS/Linux
# or: venv\Scripts\activate  # Windows

# Install dependencies
pip install -r requirements.txt
```

### 2. Set Up Azure Cognitive Search

#### Via Azure Portal

1. Go to https://portal.azure.com
2. Create resource → **Azure Cognitive Search**
3. Fill in:
   - **Resource Group**: `questionnaire-rag-rg` (create new)
   - **Service Name**: `questionnaire-search` (globally unique)
   - **Location**: `East US` (or your region)
   - **Pricing Tier**: **Standard** (minimum for semantic ranking)
4. Click "Review + Create" → "Create"
5. Wait 5-10 minutes for deployment

#### Get Credentials

1. Go to your search service in Portal
2. Click **Keys** → Copy:
   - **Primary Admin Key** → `AZURE_SEARCH_API_KEY`
   - **Endpoint URL** → `AZURE_SEARCH_ENDPOINT`

#### Create Index

Use the schema from [docs/azure_index_schemas.md](../azure_index_schemas.md):

```bash
curl -X PUT https://questionnaire-search.search.windows.net/indexes/caiq_questions?api-version=2024-07-01 \
  -H "Content-Type: application/json" \
  -H "api-key: YOUR_ADMIN_KEY" \
  -d @docs/index_schema_caiq.json
```

### 3. Configure Environment

```bash
# Copy template (if exists)
cp .env.example .env

# Edit .env with your credentials
nano .env  # or your editor
```

**Required Variables**:

## Architecture

### Components

```
QuestionaireRAG v4
├── app/core/
│   ├── azure_search.py          Azure Search client wrapper
│   ├── indexer.py               Indexing to Azure CS (hybrid)
│   ├── retriever.py             Retrieval from Azure CS
│   ├── embedder.py              OpenAI embeddings & summarization
│   └── ingestor.py              PDF/XLSX parsing
├── app/api/
│   └── routes.py                FastAPI endpoints
├── scripts/
│   └── migrate_caiq_to_azure.py  Data migration (Chroma → Azure CS)
└── tests/
    └── test_azure_integration.py  Integration tests
```

### Query Flow

```
Customer Query Request
    ↓
Load customer PDFs (ingestor.py)
    ↓
Summarize with GPT-4o (embedder.py)
    ↓
Embed summary (embedder.py → Azure OpenAI Ada-002, 1536-dim)
    ↓
Hybrid search on Azure CS (azure_search.py)
    ├─ Vector search (HNSW, cosine similarity)
    ├─ BM25 keyword search
    └─ Optional semantic ranking (AI-powered reranking)
    ↓
Enrich with full metadata (questions_store.json)
    ↓
Return top-K ranked questions with scores
    ↓
✓ Complete (target: < 2 seconds)
```

### Hybrid Search Explained

**Azure Cognitive Search hybrid search** combines two retrieval methods in a single query:

1. **Vector Search** (Semantic):
   - Finds questions semantically similar to the query
   - Uses 1536-dimensional embeddings from Ada-002
   - HNSW algorithm with cosine distance
   - Good for understanding intent

2. **BM25 Search** (Keyword):
   - Finds questions with keywords matching the query
   - Uses Azure's native BM25 analyzer
   - Good for exact term matching

3. **Score Combination**:
   - Azure combines both signals into a hybrid score (0-4 range)
   - No manual fusion needed (unlike v3's RRF)

4. **Semantic Ranking** (Optional):
   - Takes hybrid-ranked results
   - Re-scores using AI semantic understanding
   - Improves relevance (e.g., "firewall" ranks higher for network security)
   - Adds ~$5 per 1000 queries cost

### Performance Comparison

| Metric | v3 (Chroma+BM25) | v4 (Azure CS) | Improvement |
|--------|-----------------|---------------|-------------|
| Queries per search | 2 (dense + sparse) | 1 (hybrid) | ~50% fewer API calls |
| Ranking method | Manual RRF | Native Azure | Simpler, cleaner code |
| Relevance ranking | Limited | Vector + BM25 + Semantic | Better results |
| Scalability | Manual management | Azure-managed | Better |
| Multi-tenancy | Single index | Per-customer indexes | Stronger isolation |
| Code complexity | High | Low | More maintainable |

#### `POST /query`

Retrieve ranked CAIQ questions for a customer.

**Request**:
```json
{
  "customer_id": "customer_1",
  "top_k": 20,
  "use_semantic_ranking": true
}
```

**Parameters**:
- `customer_id` (string, required): Customer folder name (e.g., "customer_1")
- `top_k` (integer, optional, default=20): Number of results to return (1-263)
- `use_semantic_ranking` (boolean, optional, default=true): Enable AI-powered relevance ranking

**Response**:
```json
{
  "customer_id": "customer_1",
  "context_summary": "Organization has implemented security policies including access control, encryption, and incident response procedures. Key governance areas include data classification, third-party risk management, and compliance monitoring.",
  "total_results": 20,
  "questions": [
    {
      "rank": 1,
      "question_id": "IAM-01.1",
      "domain": "IAM",
      "question": "Does the organization have an Identity and Access Management (IAM) policy?",
      "score": 4.235,
      "semantic_score": 0.956,
      "source": "CAIQ"
    },
    {
      "rank": 2,
      "question_id": "IAM-02.1",
      "domain": "IAM",
      "question": "Does the organization enforce multi-factor authentication?",
      "score": 3.891,
      "semantic_score": 0.912,
      "source": "CAIQ"
    }
    // ... more results
  ]
}
```

**Response Fields**:
- `rank`: Position in results (1-based)
- `question_id`: CAIQ question identifier (e.g., "IAM-01.1")
- `domain`: Security domain (IAM, DATA, INFRA, etc.)
- `question`: Full question text
- `score`: Hybrid search score (vector + BM25, 0-4 range)
- `semantic_score`: Semantic ranking score (0-1, only if `use_semantic_ranking=true`)
- `source`: Data source ("CAIQ")

## Differences from v3

### What Changed

✅ **Indexing**:
- v3: Chroma DB + BM25 index
- v4: Single Azure CS hybrid index (vector + BM25 native)

✅ **Search**:
- v3: Two separate queries + manual RRF fusion
- v4: One hybrid query (Azure does the fusion)

- v3: Vector similarity + BM25 scoring

- v3: `rrf_score` field

## Troubleshooting

### Azure Credentials Not Found

**Error**: "Azure Cognitive Search credentials not found in environment"

**Solution**:
1. Check `.env` has `AZURE_SEARCH_ENDPOINT` and `AZURE_SEARCH_API_KEY`
2. Restart server
3. Verify credentials are correct in Azure Portal

### Connection Refused

**Error**: "Failed to connect to Azure Cognitive Search"

**Solution**:
1. Verify Azure service is running (Portal → status green)
2. Check network connectivity: `ping your-service.search.windows.net`
3. Verify service URL is correct

### No Results Returned

**Error**: Query returns 0 results

**Solution**:
1. Check index has documents (Portal → Indexes → Document count > 0)
2. Verify `questions_store.json` exists and is populated
3. Run health check: `curl http://localhost:8000/health`

### Latency Too High

**Error**: Queries take > 3 seconds

**Solution**:
1. Check Azure CS search units (Portal → Scale)
2. Consider disabling semantic ranking for speed
3. Check network latency to Azure
